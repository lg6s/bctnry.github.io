Setting up the environment

Setting up the environment is pretty straight-forward.
  • Composer: Follow the steps at https://getcomposer.org/download/, rename the file "composer.phar" to "composer" and put it under your $PATH.
  • Laravel: Follow the steps at https://laravel.com/docs/5.6/installation.

I've learned enough MVC stuff to know how to use one without knowing about the implementation details:
  • "Models" represent stuff. No special folder for storing "models" in laravel though, but will live with it somehow.
  • "Views" are your displaying templates.
  • "Controllers" are how you're gonna display your stuff / handle stuff from user input.
  • "Routes" are how you're gonna map URLs to different stuff.
