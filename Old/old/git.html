Quick git lookup

I learned git for a couple of hours yesterday & I forget 1/2 of them today. To keep myself from forgetting I decided to make up this page.

Create a git project

use 

git init
in your project's directory.

Cloning an existing project

use 

git clone [location] [local name]
to fetch a repo.

Working & updating

To make changes to a git repo, you need:

  • Do your work in your project's directory.
  • Use 
    git add [filename1] [filename2]...
    to declare the files that you want to update.
  • And then use 
    git commit -m "[commit message here]"
    to push a commit.

Inspecting changes

You can use 

git status
to see the changes between the git repo and your local version. Files whose name is printed in red is files whose changes haven't tracked by git. Use 
git add [filename]
and its change will be tracked from now. Use 
git diff [filename]
to see the changes of a tracked file.

Viewing the history

  • Use 
    git log
    to see all the commits on the git repo.
  • Use 
    git show HEAD
    to see the most recent commit (which is named as the HEAD commit)

Changing the history

When you changed a file by mistake, use 

git checkout HEAD [filename]
to retrieve the most recent version of the file on the repo. If you've also put the file in the staging area by mistake, use 
git reset HEAD [filename]
to cancel the add. Use 
git reset [first 7 characters of the SHA key of the commit]
to reset the repo to that commit.

Branches

Use 

git branch
to view the branches of the repo. The asterisk indicates which branch you're currently working on. Use 
git branch [branch name]
to create new branches, use 
git checkout [branch name]
to switch between branches, and use 
git branch -d [branch name]
to delete a branch. Use 
git merge [branch name]
to merge a branch to master. If you meet a merge conflict, just open the file that causes the conflict, solve the conflict manually, and just add & commit as usual.

Remotes

Roughly different versions. The origin remote is just called as "origin". Use 

git remote -v
to list all the remotes. Use 
git fetch
to get any possible changes of the repo. The changes will be put in a seperate branch whose name is "[remote name]/[branch name]". Your local copy of master is always named as "master"... Use 
git push [remote name] [branch name]
to push your work to the remote.
